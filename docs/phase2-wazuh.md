# Phase 2 — Wazuh Deployment (EDR)

## Overview

This phase covers the full deployment of Wazuh as the EDR (Endpoint Detection & Response) solution for the lab. Wazuh Manager is installed on a dedicated Ubuntu Server, and the Wazuh Agent is deployed on the Ubuntu Desktop target machine. All security events are collected, processed, and visualized through the Wazuh Dashboard.

---

## Architecture

```
Ubuntu Desktop (192.168.10.20)
        │
        │  Wazuh Agent (events, alerts, FIM)
        ▼
Ubuntu Wazuh Manager (192.168.10.30)
        │
        │  Wazuh Indexer (OpenSearch)
        ▼
  Wazuh Dashboard (https://192.168.10.30)
```

## Step 2 — Install Wazuh Manager

### Download Installation Files

```bash
wget https://packages.wazuh.com/4.11/wazuh-install.sh
wget https://packages.wazuh.com/4.11/config.yml
chmod +x wazuh-install.sh
```

### Configure config.yml

```bash
nano config.yml
```

Replace all content with:

```yaml
nodes:
  indexer:
    - name: node-1
      ip: "192.168.10.30"
  server:
    - name: wazuh-1
      ip: "192.168.10.30"
  dashboard:
    - name: dashboard
      ip: "192.168.10.30"
```

### Run Installation (Step by Step)

**Generate certificates:**
```bash
sudo bash wazuh-install.sh --generate-config-files
```

**Install Wazuh Indexer:**
```bash
sudo bash wazuh-install.sh --wazuh-indexer node-1
```

**Start the cluster:**
```bash
sudo bash wazuh-install.sh --start-cluster
```

**Install Wazuh Server:**
```bash
sudo bash wazuh-install.sh --wazuh-server wazuh-1
```

**Install Wazuh Dashboard:**
```bash
sudo bash wazuh-install.sh --wazuh-dashboard dashboard
```

> ⚠️ Each step takes 3–10 minutes. Wait for each to complete before running the next.

### Save Credentials

At the end of the installation, the installer outputs:

```
User: admin
Password: <auto-generated>
```

> Store these credentials securely. Do NOT commit them to GitHub — add a `credentials.md` file to `.gitignore`.

## Step 4 — Install Wazuh Agent (Ubuntu Desktop Target)

Run these commands on the **Ubuntu Desktop target (192.168.10.20)**:

```bash
wget https://packages.wazuh.com/4.11/apt/pool/main/w/wazuh-agent/wazuh-agent_4.11.2-1_amd64.deb
sudo WAZUH_MANAGER='192.168.10.30' WAZUH_AGENT_NAME='ubuntu-target' dpkg -i wazuh-agent_4.11.2-1_amd64.deb
```

Start and enable the agent:

```bash
sudo systemctl daemon-reload
sudo systemctl enable wazuh-agent
sudo systemctl start wazuh-agent
sudo systemctl status wazuh-agent
```

Verify the agent appears as **Active** in the dashboard:

**Wazuh Dashboard → Agents → ubuntu-target → Status: Active ✅**

## Step 5 — Verify Alert Generation

### Enable Linux Audit Logging on Target

```bash
sudo apt-get install auditd -y
sudo systemctl enable auditd
sudo systemctl start auditd
```

### Generate Test Events

Run these commands on the **Ubuntu Desktop target** to trigger alerts:

**File Integrity Monitoring (FIM):**
```bash
sudo touch /etc/test-wazuh-fim
sudo rm /etc/test-wazuh-fim
```

**Failed SSH login attempts:**
```bash
ssh invaliduser@192.168.10.20
ssh invaliduser@192.168.10.20
ssh invaliduser@192.168.10.20
```

**Suspicious commands:**
```bash
sudo cat /etc/shadow
whoami
id
```

### Verify Alerts in Dashboard

Navigate to:
**Agents → ubuntu-target → Threat Hunting → Events tab → Last 24 hours**

Expected alerts:
- Rule 554 — File added to the system
- Rule 5503 — SSH authentication failed
- Rule 5402 — Successful sudo execution

## Troubleshooting

### Wazuh Indexer fails to start
```bash
# Create missing log directory
sudo mkdir -p /var/log/wazuh-indexer
sudo chown -R wazuh-indexer:wazuh-indexer /var/log/wazuh-indexer
sudo chmod 755 /var/log/wazuh-indexer
sudo systemctl start wazuh-indexer
```

### Dashboard shows no alerts
```bash
# Check alerts are being generated
sudo wc -l /var/ossec/logs/alerts/alerts.log

# Check indexer is receiving data
curl -k -u 'admin:YOUR_PASSWORD' 'https://192.168.10.30:9200/wazuh-alerts-4.x-*/_count'
```

### Agent not connecting
```bash
# Check agent status on target
sudo systemctl status wazuh-agent

# Check manager is reachable from target
ping -c 3 192.168.10.30
telnet 192.168.10.30 1514
```

## Key Concepts Learned

**EDR (Endpoint Detection & Response)** — Wazuh monitors endpoints in real time, collecting logs, detecting anomalies, and triggering alerts based on a ruleset of 3,000+ built-in rules.

**File Integrity Monitoring (FIM)** — Wazuh tracks changes to critical files and directories (e.g. `/etc`, `/bin`), alerting when files are created, modified, or deleted.

**OSSEC Ruleset** — Wazuh's detection engine uses XML-based rules. Each rule has an ID, severity level (1-15), and a description. Rules above level 7 generate alerts visible in the dashboard.

**Wazuh Indexer** — Built on OpenSearch, it stores and indexes all alerts, enabling fast search and visualization through the dashboard.
